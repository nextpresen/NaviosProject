# NaviOs 要点サマリー

## 1. このプロジェクトが目指すもの
- **NaviOs（navios.life）**は、地域で「今日を安心して生きる判断」を支える生活ナビアプリ。
- コンセプトは、派手さよりも「地味に助かる情報」を最短で届けること。
- 住んでいる地域だけでなく、旅先でもその場で使える実用情報を提供する。

## 2. ユーザー価値（何が嬉しいか）
- 近所の生活直結情報を1か所で確認できる。
- 制度情報（補助金・支援制度）は、条件入力で「使える可能性がある制度」だけを表示。
- 必要な情報を絞って見せることで、調べる負担を減らす。

### 実装カテゴリ（優先順）
1. 実装No.1: 旅先サポート（旅先で今使えるイベント・周辺情報を即時取得）
2. 実装No.2: 地域アクティビティ（朝市・夕市・地域イベント・商店街企画）
3. 実装No.3: 生活インフラ（ゴミの日、断水、工事、役所情報）
4. 実装No.4: 日常支出最適化（特売、農家直売、移動販売）
5. 実装No.5: 行政支援ナビ（補助金、申請窓口、必要書類）

## 3. 社会背景との適合
- 制度の複雑さ、自己責任化、高齢化、地域分断といった課題に対し、
  NaviOsは「判断支援」と「地域情報の再接続」で応える設計。
- 直接的に社会課題を解決するというより、地域の暮らしや魅力を可視化し、
  「この地域で暮らせる実感」を高めることで地域離脱の抑制に寄与する。

## 4. 開発ロードマップ
- **Phase1**: 管理画面、投稿フォーム、公開状態管理、並び替え基盤。
- **Phase2**: チャット、距離順フィルタ、複数ピン、今日開催フィルタ。
- **Phase3**: 行政支援レイヤー統合（生活OS化）。

## 5. 収益化の方向性
- 自治体向け導入（B2G）
- 地域店舗の月額掲載
- 将来は医療・保険API連携も検討
- 方針: ユーザー体験を壊さない「いやらしくない収益化」

## 6. 技術方針（現時点）
- MVP優先構成: TypeScript / Tailwind CSS / Next.js(App Router) / Vercel / Supabase
- 安定後検討構成: Cloudflare Workers / Cloudflare D1
