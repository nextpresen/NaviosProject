# リファクタリング実施チェックリスト

最終更新: 2026-02-16

## 1. 事前確認

- [ ] 対象スコープをIssue/メモで明文化した
- [ ] 非対象（今回やらないこと）を明文化した
- [ ] 影響範囲ファイルを列挙した

## 2. 容量・運用

- [ ] `.next` をクリーンしてから計測した
- [ ] `du -h --max-depth=2` で容量内訳を記録した
- [ ] `node_modules` の再インストールが必要か判断した

## 3. 実装

- [ ] 関連ロジックを共通化し、重複を減らした
- [ ] public API（props/type/route）の変更点を明記した
- [ ] 既存挙動を壊さないためのフォールバックを実装した

## 4. 検証

- [ ] `npm run build` 成功
- [ ] 主要フロー手動確認（一覧 / 投稿 / 編集 / ログイン）
- [ ] エラー系動作確認（DB未初期化・認証切れなど）

## 5. ドキュメント

- [ ] 設計書へ変更目的・影響範囲を追記した
- [ ] DB変更がある場合、`docs/database` を更新した
- [ ] 次回作業のTODOを残した

## 6. 完了条件

- [ ] 差分がテーマに対して最小である
- [ ] 追加したコードに説明責務（命名/コメント/構造）がある
- [ ] ロールバック手順が明確
