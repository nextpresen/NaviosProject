# 01_はじめに

最終更新: 2026-02-16

## このフォルダについて

Naviosプロジェクトの概要と、開発を始めるための基本情報が入っています。

**初めての方は、このフォルダから読み始めてください。**

## ファイル一覧

### プロジェクト概要.md

Naviosとは何か、誰が使うのか、現状はどうなっているかを説明したドキュメントです。

**内容:**
- Naviosの説明
- 主な機能
- 利用者像
- プロジェクトの現状
- 技術スタック（簡単な説明）

**こんな人におすすめ:**
- プロジェクトに初めて参加する人
- Naviosが何をするアプリか知りたい人
- 全体像を素早く把握したい人

### 用語集.md

プロジェクトで使われる専門用語を、初心者にも分かりやすく説明したドキュメントです。

**内容:**
- アプリケーション用語（イベント、カテゴリ、タグなど）
- 技術用語（フロントエンド、API、データベースなど）
- プロジェクト用語（Phase、リファクタリングなど）
- 開発ツール用語（Git、npm、ビルドなど）

**こんな人におすすめ:**
- 技術用語が分からない人
- ドキュメントを読んでいて知らない言葉が出てきた人
- 開発経験が少ない人

## 読む順番

1. **まず**: [プロジェクト概要.md] を読む
2. **次に**: [用語集.md] で知らない用語を確認
3. **そして**: [02_設計書/] や [04_開発ガイド/] へ進む

## よくある質問

### Q1. Naviosって何？
A. 地域の「今使える情報」を地図上で見つけたり投稿したりできるWebアプリです。
詳しくは [プロジェクト概要.md] を読んでください。

### Q2. 開発を始めたいんだけど、どうすればいい？
A. [04_開発ガイド/環境構築ガイド.md] を読んで、開発環境を構築してください。

### Q3. 専門用語が分からない
A. [用語集.md] を参照してください。それでも分からない場合はチームに質問してください。

### Q4. データベースの構造を知りたい
A. [03_データベース/database設計.md] を読んでください。

### Q5. 最近何が変わったの？
A. [05_改修履歴/] を参照してください。

## 次のステップ

### 開発者として参加する場合
1. [プロジェクト概要.md] で全体像を把握
2. [用語集.md] で用語を確認
3. [04_開発ガイド/環境構築ガイド.md] で環境構築
4. [02_設計書/Phase1設計書.md] で詳細を理解

### プロジェクトマネージャーの場合
1. [プロジェクト概要.md] で全体像を把握
2. [02_設計書/Phase1設計書.md] で設計を理解
3. [05_改修履歴/] で開発進捗を確認
4. [06_リファクタリング/] で技術負債を把握

### デザイナーの場合
1. [プロジェクト概要.md] で全体像を把握
2. [05_改修履歴/Phase1_Vol1_UI_UX改修.md] でUI/UX改修内容を確認
3. 実際のアプリ（`npm run dev`）を触ってみる

分からないことがあれば、プロジェクトチームに気軽に質問してください。
